# 曲政

## 叠甲声明 (Disclaimer)

这不是一份用来炫耀的“功劳簿”，而是一个工程师的**“认知迭代数据库”**。

它记录了一个工程师的**觉醒之路**：
从早年受命打破专利而“为了不同而不同”的青涩碰撞，
到潜心通过只鳞片爪（备件维修手册）逆向国际标杆的深度积淀，
再到如今**致力用代码和计算重构机械设计**的笃行。

帮我讲出完整的故事，让我觉得生活更有意义，仅此而已。

## 我的工程主张 (Engineering Philosophy)

*在食品机械行业，承认差距是追赶的第一步。但如何追赶，决定了我们能走多远。*

*   **关于逆向工程**：
    *   测绘是必要的手段，但绝不是目的。
    *   我不满足于仅仅复刻图纸。我的目标是**透过尺寸看逻辑**——用计算去反推原厂的“Why”，把每一次测绘都变成一次深度的正向学习。
*   **关于试错必要**：
    *   物理世界的试错成本高昂。
    *   我主张**“计算驱动设计”**：能算出来的，绝不靠猜；能用代码模拟的，绝不急着加工。我们省下的每一次试错成本，都应该转化为团队的经验文档。
*   **关于现场主义 (Gembaism)**：
    *   **“做一做才知道”**：这是我的恩师崔文豪教会我的。这里的“知道”不是书本知识，而是面对真实物理世界复杂性时的身体记忆。
    *   **底气**：三年的装配一线经历，让我练就了一种直觉——装配调定后随手画图记录百分表数值。这种对“微米级变化”的感知，是我如今做设计的隐形尺子。
*   **关于工具**：
    *   Python 和软件工程思维，是现代机械设计师的“第二大脑”。

## 专业作品与沉淀 (Key Works)

*以下案例见证了我“从测绘到计算，从仿制到重构”的路径。*

- **2024.04 [深度重构] G.Mondini Trave 350 国产化项目**
  - **思考**：面对复杂的封口顶升机构，盲目照搬会导致国产件水土不服。
  - **行动**：建立了完整的运动学计算模型，验证了国产伺服与凸轮的匹配度。不仅交付了样机，更沉淀了《关键机构设计备忘录》，让后人“有据可查”。
- **2019.10 [算法仿真] 利乐柔性机双夹爪电子凸轮**
  - **突破**：参考 Adams 会议资料，用 Python 编写算法重现了原厂复杂的凸轮曲线。这是我第一次意识到：**代码可以让机械设计变得如此精准和优雅。**
  - **代码**：[GitHub - tetracamthon](https://github.com/John-Qu/tetracamthon)
- **2025.08 [效率工具] Solidworks 嵌套 BOM 自动化**
  - **理念**：机械设计不应被繁琐的表格工作淹没。利用 Python 自动化处理 BOM，践行软件工程的 DRY (Don't Repeat Yourself) 原则。
  - **代码**：[GitHub - John-Qu/ExportNestedBOM](https://github.com/John-Qu/ExportNestedBOM)
- **2023.01 [正向设计] 电池测量工装 (FreeCAD)**
  - **实践**：在极其受限的空间内，利用全参数化设计一次性解决了干涉问题。证明了即使是小工装，计算也能带来巨大的确定性。
- **2009~2010 [逆向推演] Tetra Pak A3-Flex 机型全参数重构**
  - **背景**：在经历了早期“为了创新而创新”的挫败后，我意识到差距，主动转向深度学习。
  - **挑战**：在没有实物、只有 SPC（备件手册）和 MM（维修手册）图片的情况下。
  - **行动**：依靠工程直觉和图片比例分析，**“猜”出了**关键尺寸与结构逻辑，成功在 CAD 中重构了 A3-Flex 的核心夹爪与驱动部分。
  - **收获**：这段“盲人摸象”却拼出全貌的经历，重塑了我的设计观——**结构必须服从于逻辑，而非为了标新立异。**
- **2008.10 [早期尝试] 利乐高速机终端排包机构研发**
  - **复盘**：研究生时期，参与打破专利壁垒的项目。虽然做出了实物，但因缺乏经验，陷入了“为了规避专利而设计”的误区，导致成本与性能不占优。
  - **意义**：这次不完美的尝试是我职业生涯的第一课，它让我明白了**正向设计的核心是“更优”，而不是“不同”。**
